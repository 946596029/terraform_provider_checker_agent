# Markdown 检查规则的思路

1. 先写在文件里，写成一个脚本
2. 结构，先解析 Markdown 文档为树状 AST
3. 遍历树状 AST，提取这段代码，检测到特征
4. 先走传统流程，再走 AI 流程

要注意的点:
  - AI不可靠是设计决定的，无法避免小概率失误
  - 代码检查不是直接跳过人进行代码的修改
    - 检查出不符合代码规范的地方，需要人来再看一遍
    - 检查时间变短
    - 检查准确性增强
