# 开始

## 原则
1. 使用最小路径实现效果
2. 在验证初期demo后，需要寻找最小可行一个产品路径
    保证可靠，投入人力小的功能点开始迭代

## 路标
#### 基础工具能力
- 1. 调用 LLM 接口
- 2. 使用 token 计数，判断是否超过 LLM 上下文限制
- 3. 使用 prompt 来与 LLM 对话
- 4. 进行一次多轮对话
- 5. 使用 prompt 结构化模板来使 prompt 更加灵活
- 6. 使用 JSON 进行存储，使用 Markdown 发送给 LLM
#### 初步验证demo
- 1. 使用 prompt 模板进行一次全量调用，看一下效果
- 2. 记录全量调用出现的问题
#### 对接mcp工具
- 1. 使用 mcp 工具来访问代码文件
